## 修复：手机本地打开白屏问题
- 根因：Via 等手机浏览器用 file:// 协议打开 HTML 时禁用 localStorage，State.init() 一上来访问 localStorage 抛异常导致整个脚本崩溃白屏
- 修复1：state.js 新增 testStorage() 探测，所有 localStorage 读写包 try/catch，不可用时降级为内存存储（memStore），当次会话仍可玩
- 修复2：index.html 的 Service Worker 注册加 protocol 判断，file:// 下跳过（SW 在 file:// 下无法注册）
- 修复3：难度选择页显示存档状态提示（✓ 支持 / ⚠ 不支持 + 建议）
- 关键教训：手机浏览器 file:// 协议限制多，localStorage/ServiceWorker/部分 fetch API 都可能不可用，做 H5 游戏必须做降级处理

### 需求
- 用户在产业页买了大豆后，想把小麦上的员工直接调到大豆，不想去员工页先撤回再分配

### 实现
- **employees.js**：新增 `transfer(fromGroupId, n, toType, toCategory)` 方法，从已分配组拆 n 人直接转移到另一个品类（同等级合并）
- **staff.js**：重写 `showAssignPickerByIndustry()` 为完整员工管理弹窗：
  - 当前员工区：显示已派到此品类的员工组，每条有"撤回"按钮
  - 可调入区：分两组显示
    - 未分配员工（"调入"按钮 → showQtyPicker → assign）
    - 同产业其他品类的员工（"调拨"按钮 → showTransferPicker → transfer）
- **staff.js**：新增 `showTransferPicker()` — 关闭弹窗后开 numberPicker 选调拨数量
- **staff.js**：`showUnassignPicker()` 改为先 closeModal 再 setTimeout 开弹窗（避免从产业弹窗内调用时弹窗叠加）
- industry.js 不需要改，"调整/派人"按钮已调用 showAssignPickerByIndustry

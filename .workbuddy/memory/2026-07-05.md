## 修复：新闻按钮无反应 + 时间快进卡死

### 根因
- `State.startGame()` 初始数据中遗漏了 `news: []` 字段 → `State.data.news` 为 `undefined`
- `State.migrate()` 也没有补全 `news` 字段
- 新游戏开始后，`Engine.advance()` 中 `State.data.news.unshift(...)` 抛 `TypeError: Cannot read properties of undefined (reading 'unshift')`
- 异常导致 `Home.advance()` 中 `_advancing` 永远卡在 `true`，后续所有快进按钮被 `if (this._advancing) return` 拦截
- 新闻页 `renderNews()` 中 `news.length` 同样抛 TypeError → 页面渲染崩溃 → 点击新闻按钮"没反应"

### 修复内容
1. **state.js `startGame`**：初始数据添加 `news: []`
2. **state.js `migrate`**：添加 `if (!d.news) d.news = [];` 补全旧存档
3. **engine.js `getNewsTags`**：添加 `if (!news || !news.effects) return tags;` 防御性检查
4. **engine.js `advance`**：`State.data.news.unshift` 前加 `if (!State.data.news) State.data.news = [];` 防护
5. **time.js `advanceDay`**：同上防护
6. **home.js `advance`**：用 try-catch 包装 `Engine.advance`，异常时重置 `_advancing` + `TimeManager` + toast 报错
7. **home.js `advance`**：结算弹窗设置 `dataset.dismissable = 'false'`，禁止点击外面关闭，确保只能通过"关闭"按钮关闭（`_dismissAdvanceModal` 重置 `_advancing`）
8. **sw.js**：缓存版本升至 v10

### 次要 bug（一并修复）
- 快进弹窗如果通过点击弹窗外面关闭（而非"关闭"按钮），`_advancing` 不会被重置 → 后续快进永久失效
- 修复：结算弹窗禁止点击外面关闭，只能点"关闭"按钮

### 教训
- 新增功能字段时必须同步更新 `startGame` 初始数据和 `migrate` 补全逻辑
- 异步操作（弹窗回调）的状态标志重置必须覆盖所有关闭路径

## 修复：主页农业/矿业/冶金/工厂点击无反应 + 返回按钮要点多次

### 根因
- `js/pages/industry.js` 和 `js/pages/metals.js` 两个文件中文全是乱码（UTF-8/GBK 编码损坏）
- 乱码吞掉了字符串闭合引号（如 `'贵金属'` 变成 `'璐甸噾灞?`，`'—'` 变成 `'鈥?`），导致 JS 语法错误
- `industry.js` 有语法错误 → `Pages.industry` 未定义 → 农业/矿业/冶金/工厂/地产 5 个按钮（都跳转到 `industry` 页）点击无响应
- `metals.js` 同理 → 贵金属按钮也会坏
- **返回按钮要点多次的原因**：点击坏掉的产业卡片时，`Router.go` 已把页面压入历史栈，但 `render()` 抛异常、界面仍停留主页；继续点继续压栈，历史栈堆满 `industry`，每次 `back()` 只弹一个，需多次才能清空回到真正上一页

### 修复
- 用正确 UTF-8 中文重写 `industry.js`（保留全部逻辑：产出/消费/配方/升级/出售/购入）
- 用正确 UTF-8 中文重写 `metals.js`（保留买入/卖出/手续费/持仓逻辑）
- `node --check` 验证 16 个 JS 文件全部语法正常，全项目无残留乱码

### 教训
- 文件出现乱码时，乱码可能破坏 JS 语法（吞掉引号），不只是显示问题
- "点击无反应 + 返回要点多次"组合症状 = 目标页脚本加载失败导致 render 抛异常 + 历史栈被无效跳转污染
- 排查此类问题：`node --check` 逐文件语法检查 + grep 扫描乱码字符，能快速定位

## 修复：深色模式下文字黑色看不清

### 问题
- 深色主题下，卡片标题（银行/股票/基金/产业等）、卡片数值（¥0等）、列表值均为黑色，在深色背景上几乎不可见
- 根因：`.card-title`、`.card-value`、`.list-value` 三个类未设置 `color` 属性，依赖 body 继承；但部分浏览器/场景下继承链断裂

### 修复
1. **CSS**：`.card-title` / `.card-value` / `.list-value` 全部添加 `color: var(--text-primary)`
2. **body** 已在之前改为 `var(--text-primary)`（浅色 `#1a1a1a` / 深色 `#f0f0f3`）
3. **SW 缓存版本** v13 → v14

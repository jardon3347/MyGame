# 项目记忆 - 集团模拟经营文字游戏

## 项目概述
手机可玩的文字类集团模拟经营游戏。主界面分金融/实业两类按钮，点击跳转子页面，底部固定总览栏。

## 技术栈
- H5 单页应用（HTML/CSS/JS）
- PWA 配置（可加桌面、离线可玩）
- Capacitor 打包成 Android APK（后续）
- localStorage 存档
- 部署：CloudStudio 云端部署，手机扫码体验

## 设计规范
- 视觉风格：简洁卡片风（白底卡片、0.5px边框、圆角、系统字体）
- 移动端优先：竖屏布局、按钮触摸区≥44px、大字号宽松行距
- 中国股市惯例：涨红跌绿、¥货币符号

## 板块结构
- 金融：银行（存款/贷款）、股票、基金、贵金属
- 实业：农业（产能型）、矿业、冶金（产能型）、工厂（产能型）、地产、**物流**、员工、仓库
- 底部总览：集团概览、银行、新闻

## 时间设定
- 起始日期：2018年1月1日（周一）
- 推进方式：玩家点击驱动或自然流逝（10分钟/天），非实时
- 加速档位：×1（看结算明细）/ ×7（遇事件暂停）/ ×30（快进）
- 新闻事件系统：真实经济史事件 + 随机模板，3-5天触发一次

## 产业系统

### 产能型 vs 非产能型
- **产能型**（farm/metall/factory）：无 `cost`，依赖地产提供产能容量，通过容量滑块分配。无升级功能，日收入已翻倍平衡。
  - `_isCapacity()` 判断
  - `landPrereqs` 配置关联地产类型
  - `capacityPerLand` 配置每套地产提供的产能数
  - 操作按钮：减少 / 增加（通过 `allocMinus`/`allocPlus`）
- **非产能型**（mining/estate/logistics）：有 `cost`，直接购买，支持升级。操作按钮：出售 / 升级 / 加购

### 工厂产品系统（100个产品）
- 10个工厂 × 10个产品（basic/mid/high 三级）
- 产品消耗仓库原料，产出成品到仓库
- 玩家在工厂卡片内通过 +/- 按钮分配生产线
- `FactoryProducts` 模块管理产品定义、配方、产出、售价
- 每日结算阶段3：消耗原料 → 产出成品 → 现金收入

### 物流系统（方案C，2026-07-06 实现）
- 5个物流站：基础/区域/智能/冷链/跨境
- 地产前置：需先购「物流产业园」
- 规则面板：物流产业页下方显示，支持添加/删除规则
- 规则类型：自动卖出（超过阈值）、自动买入（低于阈值）
- 等级加成：每级+20%槽位、手续费降低
- 每日结算阶段5自动执行规则

### 地产类型
- 住宅（员工容量）、仓库（原料存储）、农用地、采矿权、工业用地、**产业园**（工厂前置）、**物流产业园**（物流前置）

## 员工系统
- 按类分组，4个等级，分配到产业
- 员工加成影响产出，无员工则无产出
- 按钮颜色区分：已派员工→蓝色（info）、未派员工→橙色（warning）

## 每日结算流程（engine.js advanceOneDay）
1. 矿业+农业产出原料进仓库
2. 冶金消耗矿石→产出金属+现金
3. 工厂消耗原料→产出成品+现金（支持产品系统）
4. 其他产业现金收入
5. **物流自动买卖规则执行**
6. 员工薪水、存款利息、贷款利息、维护成本
7. 股价/金属价/原料价波动
8. 日期推进、新闻事件触发

## 重要 Bug 修复记录

### logisticsRules 类型不匹配（2026-07-06）
- `state.js` 初始化 `logisticsRules: {}`（对象）但 `logistics.js` 全部按数组使用 → `forEach` 崩溃 → 日期推进完全失败
- 修复：统一为 `[]`，migrate 兼容旧存档

### 产业页 tab 点击无反应（2026-07-06）
- onclick 写 `Industry.switchTab(...)` 但 `switchTab` 实际在 `Pages.industry` 上
- `_renderDualTabs` 缺少"我的产业"内容 div
- 修复：onclick 改为 `Pages.industry.switchTab(...)`，补全 tab-content 结构

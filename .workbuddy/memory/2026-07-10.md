## 首次购买矿业多份时许可证费用重复计算

### 问题
`buy()` 中 `unitCost = cat.cost + cat.licenseCost`，买N份时 `totalCost = unitCost × N`，许可证费被收了N次。
例如首次买10稀土矿：原价 `¥600,000 × 10 = ¥6,000,000`（许可证收了10次），正确应为 `¥150,000 × 10 + ¥450,000 = ¥1,950,000`（许可证只收1次）

### 改动
- `buy()`：拆分为 `baseCost`（单价）和 `licenseFee`（一次性），`totalCost = baseCost × qty + licenseFee`；`maxQty` 扣除许可证费后计算
- `purchaseItem()`：显示价格改为 `cat.cost`，许可证单独标注

## ES Module 迁移（2026-07-10）

### 改动范围
全部 27 个 JS 文件改为 ES Module（import/export），新增 `js/main.js` 入口，`index.html` 26 个 `<script>` 标签缩减为 1 个 `<script type="module">`。

### 迁移策略
- 叶子节点优先：data.js → charts.js → achievements/competitors/futures/logistics → 核心层 → pages
- 保留 `window.X = X` 全局出口：HTML onclick 中的 Router.go()、UI.toast()、State.formatMoney() 等依赖全局作用域，全部保留 window 赋值
- 循环依赖通过 ESM 动态绑定处理：ui.js ↔ pages/home.js、state ↔ engine ↔ employees

### 关键文件
- `js/main.js`：统一入口，按顺序导入所有模块，末尾调用 State.init() 和 TimeManager.start()
- `index.html`：`<script type="module" src="js/main.js"></script>` 替代 27 行旧 script 标签
- `package.json`：新增 `npm run dev`（live-server）和 `npm run dev:py`（Python HTTP）开发启动脚本
- `sw.js`：缓存版本 v32→v33，追加 main.js 到缓存列表

### 注意事项
- ES Module 要求 HTTP 协议，不能再双击 index.html 打开（file:// 不支持 ESM）
- 启动方式：`npm run dev:py`（Python，无需安装）或 `npx live-server`
- FactoryProducts.init() 在 import 时自动执行，依赖 window.DATA 已就绪（data.js 先于它加载）

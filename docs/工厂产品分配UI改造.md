# 工厂产品分配 UI 改造

> 改造日期：2026-07-06
> 对应 PR/提交：industry.js 大改 + 产品分配交互系统

---

## 改造目标

将工厂从"固定配方统一产出"改为"每条生产线可分配给具体产品"的模式，实现：

- 玩家拥有工厂（如食品厂 × 10 条生产线）
- 每条生产线可分配给具体产品（如面包 × 5，饼干 × 3，糖果 × 2）
- 产品消耗对应的原料
- 产品产出成品到仓库（可手动卖出或自动结算）

## 本次改造内容

### 修改的文件

### 新增 UI 方法 — Pages.industry

#### `_renderFactoryProductCard(type, o, cat, ind, qty, empCnt, empMult, hasStaff, inv)`

工厂专属产品卡片渲染，结构：

```
┌──────────────────────────────────────────┐
│ 食品厂 · Lv1 × 10 条生产线    [调整员工] │
│ 员工 5人 · 加成 ×8.0 · 日入 ¥85,000     │
│ 总产能 10 · 已分配 8 · 剩余 2            │
│                                          │
│ ┌─ 产品分配 ─────────────────────────┐   │
│ │ 产品           产能  日入          │   │
│ │ 面包[基础]     5    ¥42,500 [+][−] │   │
│ │   配方: 小麦x1.2 · ¥85/条          │   │
│ │ 合计            8    ¥71,000       │   │
│ └────────────────────────────────────┘   │
│ ⚠ 部分产品原料不足                       │
│                                          │
│ [分配产品] [减少] [升级] [增加]          │
└──────────────────────────────────────────┘
```

关键特性：
- 每个产品行显示：产品名 + **等级标签**（基础/中端/高端）+ 配方 + 产能数 + 日收入
- 原料紧缺时显示 ⚠ 标记
- 产能全满时「剩余」显示红色
- 无员工时整个卡片显示 ⚠️ 无员工 · 无产出

#### `_renderProductRow(type, categoryCode, product, alloc, prodIncomes, canAdjust)`

单个产品行渲染，包含：
- 等级标签颜色（基础=灰色、中端=蓝色、高端=橙色）
- 配方文字（如"小麦 x1.2 + 大豆 x0.5"）
- 已分配产能数（居中数字）
- 日收入（绿色）
- [+]/[−] 快捷操作按钮

### 新增交互方法 — Industry

| 方法 | 用途 |
|------|------|
| `_ensureProductAlloc(type, cat)` | 确保工厂 state 对象有 `products` 字段 |
| `incProdAlloc(type, cat, prodCode)` | 快速 +1：为指定产品增加一条生产线 |
| `decProdAlloc(type, cat, prodCode)` | 快速 −1：从指定产品撤回一条生产线 |
| `allocProduct(type, cat)` | 打开批量分配弹窗 |
| `_allocPickInc(id, prodCode)` | 弹窗内调整 +1 |
| `_allocPickDec(id, prodCode)` | 弹窗内调整 −1 |
| `_allocSubmit()` | 提交弹窗内的分配方案并保存 |

#### 批量分配弹窗 `allocProduct()`

```
┌──────────────────────────────┐
│ 分配产品 - 食品厂             │
│ 剩余可分配: 2 条生产线       │
│                              │
│ 面包(基础)      5  [-] [+]   │
│   小麦x1.2 · ¥85/条          │
│ 饼干(基础)      3  [-] [+]   │
│   小麦x1.0·大豆x0.5·¥95/条   │
│ 方便面(中端)    0  [-] [+]   │
│   小麦x1.5·油菜x0.3·¥110/条  │
│ ...                          │
│                              │
│ [取消] [确认分配]            │
└──────────────────────────────┘
```

弹窗特性：
- 显示剩余可分配产能数
- 每个产品带基础配方和售价信息
- [+] 受工厂总产能限制（不可超额分配）
- [−] 可减到 0（完全撤回）
- 确认后保存到存档，刷新页面

### 修改统计方法

#### `render()` 日收入计算

工厂类型优先使用产品级日收入：

```javascript
if (type === 'factory' && FactoryProducts && o.products && Object.keys(o.products).length > 0) {
  daily = FactoryProducts.factoryDailyIncome(o.category);
}
```

非工厂或无产品分配时回退到旧的 `cat.dailyIncome * qty * ...` 计算。

#### `purchaseItem()` 工厂描述

可购买列表中的工厂信息从显示旧配方改为显示：

```
食品厂 · 日入 ¥208/条 · 回本 —天 · 10种产品可分配
```

## 产品收入体系

### 等级体系（Option B）

| 等级 | 配方复杂度 | 收入范围 | 策略定位 |
|------|-----------|---------|---------|
| 基础 (basic) | 1-2 种原料 | ¥50-120/条 | 稳定低风险 |
| 中端 (mid) | 2-3 种原料 | ¥130-200/条 | 平衡型 |
| 高端 (high) | 3-4 种原料 | ¥210-420/条 | 高风险高回报 |

### 原料消耗逻辑

- 每日结算时，Engine 遍历每个工厂的产品分配
- 检查仓库原料库存，按满足率折算产出
- 消耗库存原料
- 产出成品到仓库（成品可手动卖出或超额自动卖）

---

## 决策确认

三个关键决策全部选择 **Option B**：

| 决策 | 选项 | 实现 |
|------|------|------|
| 产品产出方式 | **B：成品到仓库** | `FactoryProducts.produceProductOutput()` 已有实现 |
| 配方复杂度 | **B：高端 3-4 种** | 部分产品已含 3 种原料（手机、调味品、生物肥等） |
| 收入平衡 | **B：高端高收入高消耗** | sellPrice 从 ¥50（袜子）到 ¥420（手机）差异 8 倍 |

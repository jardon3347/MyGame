## 🎮 盛世集团 · 经营模拟 — 优化建议方案

### 一、核心玩法与数值平衡

**1. 资金曲线存在"死亡谷"问题**
- 当前难度下，玩家前期靠存贷款吃利息就能维持，中期买了产业园+工厂后原料消耗大但产出慢，容易陷入"现金枯竭但资产一堆"的困境
- **建议**：增加一笔"应急贷款"或"信用透支"机制（低于零时自动贷款，利率较高），给玩家缓冲；或者在开局增加一次"政策补贴"事件

**2. 产能型产业（农业/工厂）的员工乘数影响过大**
- 无员工 = 零产出，这意味着招聘节奏完全决定了经营节奏
- **建议**：让产能型产业有一个"基础产出"（比如无员工时也有 30% 产能），员工是加速器而不是开关

**3. 工厂产品数量多但同质化严重**
- 100 个产品中大部分只是原料组合不同，玩家实际只会选利润最高的几个
- **建议**：引入"市场需求波动"机制 — 某些产品在特定新闻事件下价格翻倍（如"食品安全事件"→食品类产品涨价），迫使玩家动态调整产线分配

**4. 破产阈值 -50,000 太容易触发**
- 一旦进入负现金流区间，员工薪水+维护成本会迅速把玩家拖入破产
- **建议**：增加"裁员"和"变卖资产"的紧急操作按钮，让玩家在破产前有明确的自救手段（而不是手动去每个页面一个一个操作）

---

### 二、中后期可玩性

**5. 缺少"中期目标"**
- 玩家买完产业、分配完员工后，缺少驱动力
- **建议方案**：
  - **成就系统**：如"总资产突破 1 亿"、"持有全部 20 只股票"、"工厂 100 条产线全开"
  - **评级体系**：集团评级（从个体户→小微企业→上市公司→世界500强），不同评级解锁新功能
  - **排行榜**：虽然是单机，但可以设计"历史最高总资产"挑战模式

**6. 缺少"产业链整合"深度**
- 当前产业链是单向的（矿→冶金→工厂→卖），缺少纵向整合的奖励
- **建议**：引入"上下游协同加成" — 同时持有上游和下游产业时，中间环节成本降低 10-15%，鼓励玩家打通整条链

**7. 新闻事件影响偏弱**
- 新闻事件影响的是股价和利率，但对产业实际经营影响有限
- **建议**：增加产业类新闻效果，如：
  - "环保督查"→冶金/工厂维护成本 ×2
  - "粮食丰收"→农业产品价格 -20%
  - "芯片短缺"→电子厂产品价格 +50%
  - "疫情反复"→物流效率 -30%

### 三、UI/UX 改进

**8. 主页（概览）信息过载**
- 概览页同时展示资产分布、产业收入、新闻历史、时间条，信息密度高但重点不突出
- **建议**：
  - 增加一个"关键指标卡片"放在最上方：今日净收入 / 总资产 / 员工数，用大号数字一目了然
  - 产业收入列表可以按收入排序，默认显示 Top 5，点击展开全部

**9. 产业详情页操作路径太深**
- 要分配一个员工到工厂，需要：产业→选择工厂→调整→选择员工等级→分配
- **建议**：在每个产业卡片上增加"一键配置"快捷入口，如"派人"按钮直接弹出分配面板

**10. 缺少数据趋势可视化**
- 所有数据都是瞬时快照，看不到趋势
- **建议**：
  - 概览页增加"7日净收入折线图"（类似 K 线图的简化版）
  - 每个产业卡片显示"近7日收入趋势"（迷你 sparkline）
  - 银行存款页面增加"利率历史曲线"

**11. 仓库页体验待优化**
- 原料种类多（34种），列表很长
- **建议**：
  - 增加"智能推荐"功能：根据当前工厂消耗，自动高亮哪些原料库存不足
  - 增加"一键补货"按钮：根据工厂每日消耗量，自动买入缺少的原料至 7 天库存

### 四、技术架构建议

**12. 缺少撤销/回档机制**
- 误操作（如卖错了股票、分配错了员工）没有挽回手段
- **建议**：在 State 中维护一个"快照栈"（最多 5 步），关键操作前自动存快照，提供"撤销"按钮

**13. save/load 逻辑可增强**
- 当前每次 State.save() 都全量写入 localStorage，后期数据量大时可能有性能问题
- **建议**：
  - 增加自动存档（每 10 天自动存一次），手动存档保持
  - 增加"存档管理"页面：显示存档时间、游戏进度、允许删档

**14. 缺少统计面板**
- **建议**：新增一个"经营报告"页面（可放在概览页子 tab），展示：
  - 经营天数、总收入、总支出、净利润
  - 各产业贡献占比饼图
  - 最赚钱的产业 TOP 3
  - 历史最高资产 / 最低资产

**15. PWA 缓存策略需优化**
- 当前 sw.js 缓存版本号手动递增（v14），容易遗忘
- **建议**：改为文件内容哈希自动控制缓存失效，或者用 build 工具生成带 hash 的文件名

### 五、新功能方案

**16. 商业谈判/合约系统（中期目标）**
- 玩家可以签订"供货合约"，锁定价格批量供应给 NPC 企业
- 合约周期 30-90 天，提前解除需付违约金
- 这会增加战略深度：是押注市场价还是锁价保底

**17. 竞争对手系统**
- 加入 2-3 个 AI 竞争对手，他们也在买卖产业、参与市场
- 玩家可以看到竞争对手的公开信息（总资产排名、主要产业）
- 争夺"市场第一"的长期目标

**18. 金融衍生品扩展**
- 当前金融只有股票/基金/贵金属
- **建议**：中期增加"期货"板块（铜期货、原油期货），有杠杆和到期日，风险更高但收益更大

**19. 产业并购**
- 玩家可以合并同类型产业（如合并两家农业）获得规模效应加成
- 需要支付"整合成本"，整合后员工数合并、维护成本降低

**20. 研发系统**
- 每个产业可以投入"研发资金"提升效率
- 研发需要时间（10-30天），完成后永久提升该产业产出系数
- 高级产业（如电子厂）研发效果更好

### 六、短期优先级排序

如果你打算继续开发，我建议按这个顺序：

| 优先级 | 内容 | 理由 |
|--------|------|------|
| 🔴 P0 | 撤销/回档机制 | 防止误操作导致弃档 |
| 🔴 P0 | 破产前紧急操作面板 | 当前破产太突然，体验差 |
| 🟡 P1 | 经营报告/统计面板 | 增强"我在做什么"的感知 |
| 🟡 P1 | 7日趋势折线图 | 让数据活起来 |
| 🟡 P1 | 产业类新闻效果 | 增强新闻的策略意义 |
| 🟢 P2 | 产业链协同加成 | 增加中期深度 |
| 🟢 P2 | 成就系统 | 给玩家中长期目标 |
| 🔵 P3 | 竞争对手 AI | 大功能，后期考虑 |
| 🔵 P3 | 期货/并购/研发 | 丰富中后期内容 |
